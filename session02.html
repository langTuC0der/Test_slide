Lesson 01 · 2 tình huống thực tế

Memory Leak →

Module gợi ý AI rò rỉ bộ nhớ trong cùng JVM với Order/Payment.

  • OutOfMemoryError chiếm hết RAM server
  • Không chỉ mất gợi ý — cả đặt hàng & thanh toán tê liệt
  • Tổn thất doanh thu trong ~2 giờ khắc phục
Bài học: — lỗi 1 module = sập cả hệ thống

Scale không đồng đều

Đặt vé sự kiện: 98% xem lịch (read) · 2% mua vé (write).

  • Scale 10 instances = clone cả module Payment nặng
  • MySQL connection pool nghẽn dù CPU còn dư
  • Tài nguyên lãng phí, latency tăng
Bài học: không scale đúng chỗ = tốn tiền & vẫn chậm

Lesson 04 · Nguyên tắc & anti-pattern
No cross-DB access

Quy tắc vàng

Mỗi service sở hữu DB riêng. Truy cập từ ngoài chỉ qua API — không đọc/ghi thẳng DB service khác.

Anti-pattern:

  • • Schema coupling — đổi cột → crash service khác
  • • Connection / table lock tranh chấp
  • • Rủi ro bảo mật (đọc chéo dữ liệu nhạy cảm)

Cái giá phải trả

Không JOIN SQL chéo · không đa service · phải dùng / / Event.

Lesson 04 · Thay thế SQL JOIN phân tán
Composer · Parallel calls · Merge
